# Phase VI: Semantic Matching Plan

## 1. Overview
This phase implements the "Search" component of the system. We will leverage the neural embeddings generated by `SentenceTransformers` to find the most relevant resumes for any given job description (JD).

## 2. Technical Approach

### 2.1 Similarity Metric
We will use **Cosine Similarity**. 
- Range: [-1, 1], where 1 means identical semantic meaning.
- Why: Consine similarity measures the angle between vectors, which is invariant to the length/magnitude of the text (essential for comparing short JDs against long resumes).

### 2.2 Ranking Procedure
1.  **Input**: Raw Job Description text.
2.  **Transformation**: 
    - Convert JD to a 384-dimensional vector using `all-MiniLM-L6-v2`.
    - Apply the pre-fitted `scaler.pkl` to normalize dimensions.
3.  **Search**: Compute similarity scores against the entire `resume_embeddings.npy` database.
4.  **Category Filtering** (Optional): Use `best_classifier.pkl` to predict the category of the JD first, then optionally boost/filter resumes within that category.
5.  **Output**: Sorted list of (Resume_ID, Similarity_Score).
